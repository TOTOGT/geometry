Orthogenesis — Moon Base Geometry

Formal Verification of a Phased Moon Base

Orthogenesis models NASA’s lunar base as a mathematically rigorous honeycomb colony: axial hex coordinates, geometric growth across deployment stages, and a Colony.expand operator that mirrors NASA’s three-phase build-out — from the first 4,000 kg payload to a continuous crewed presence at the lunar South Pole.

“NASA is embarking on the most ambitious space project in recent history: building a Moon Base.”
— NASA Moon Base User’s Guide, Architecture Resources, April 2026

Why Lean 4

The NASA Moon Base Architecture Definition Document identifies functional gaps — capabilities that are unallocated or under-specified — across seven sub-architectures: Autonomous Systems, Communications & PNT, Habitation, Logistics, Mobility, Power, and Transportation. Orthogenesis treats these gaps as proof obligations. A Moon Base that compiles has no unmet structural requirements. A sorry is an open gap.

Architecture Layers

  • Orthogenesis.Geometry.HexGrid — axial coordinates, Euclidean embedding
  • Orthogenesis.Geometry.Growth — geometric radius model R(n) = g^n
  • Orthogenesis.Architecture.Cell — (HexCoord, stage) pair
  • Orthogenesis.Architecture.Colony — Finset Cell + expansion operator

Hex Grid

Modules on the lunar surface are arranged in a pointy-top axial hex grid — the same honeycomb geometry used in pressurized habitat clusters and ISRU pad layouts. Every hex has exactly six neighbors: 

def hexNeighbors (h : HexCoord) : List HexCoord :=
  [ ⟨h.q+1, h.r⟩, ⟨h.q+1, h.r-1⟩, ⟨h.q, h.r-1⟩,
    ⟨h.q-1, h.r⟩, ⟨h.q-1, h.r+1⟩, ⟨h.q, h.r+1⟩ ]

Euclidean embedding:

x = q + r/2
y = (√3 / 2) · r

Growth Model

Each cell carries a stage indexing a geometric growth sequence. Structural radius: 

def R (P : GrowthParams) (n : ℕ) : ℝ := P.g ^ n

NASA’s phased payload scaling matches this model: 4,000 kg → 60,000 kg → 150,000 kg. Growth factor g ≈ 3.87.

Colony Expansion

Colony.expand adds all six neighbors of every cell and increments stage: 

def Colony.expand (C : Colony) : Colony :=
  let newCells :=
    C.cells.fold
      (fun acc c =>
        let neigh := hexNeighbors c.coord
        let stage := c.stage + 1
        let new := neigh.map (fun h => Cell.mk h stage)
        acc ∪ new.toFinset)
      ∅
  { cells := C.cells ∪ newCells }

One call to expand = one NASA phase. Depth 2 = Phases 01, 02, 03.
